1:FlashVml(闪耀之星)3.0功能概括

FlashVml(闪耀之星)3.0是基于WEB浏览器可直接运行使用的{失量图像(2D/3D)、VmlFlash动画、VML3D网页、其他相关}可视化设计软件（免费、源码公开）。用其开发的作品，可直接编译保存为HTML、或转换为其他格式（如XML、XSL、ASP、ASP。NET格式等，转换功能为今后可行的后续版开发计划）前、后台网页文件，完全可独立运行于所有支持VML的WEB浏览器（如覆盖率最广的微软IE5+浏览器）。

4:设计宗旨

2.0 to 3.0不仅在功能、界面等方面有了大跨越式的进取，而且在其他细致入微、点滴之处也下了很大的工夫。也正印证了原作者一贯的设计宗旨：精益求精、细致入微、先用户之忧而忧。

3.0取消了2.0一些不实用的功能{
放大、缩小、基于line线条的记忆线画笔、侦式动画设计（分钟）的概念、透明度处理
}

3.0优化了2.0了一些实用的绘图、图像处理、动画功能{
3.0支持光标在网格上时直接画图，而2.0则会跳出禁止提示
1.0版右键菜单的“反转”、“颠倒”虽能用，却会使图形失真，2.0取消了这两个实用功能，3.0则又卷土重来，而且不在失真。
2.0版已封装了“裁剪、图形地震、窗口地震”流线动画特技，但却只能单进程使用（一次只能控制一个图形使用该特技）。而3.0则又新增了“3D旋转”流线动画特技，并让这3者均支持多进程（一次控制任意多个图形）
2.0侦式动画设计存在不实用的“分钟”概念，3.0则改换为“片长（影片长度）”的概念，让你设计时间长达600-3600秒任意秒的动画程序，并不影响启动速度
2.0如果绘制一个图形会生成300个字优化不全面的VML代码，那么3.0则只需200个字左右
2.0用户拖动滚动条，浮动工具栏不会跟随移动。而3.0则自动跟随
2.0给图形定义ID名称无法使用重复的ID，而3.0则可以，且自动给予ID编组、排序
2.0的3D角度旋转是数据设置+鼠标点击控制的，而3.0的3D旋转支持则是让你用鼠标随意调整
2.0的透明度处理很实用但被取消了，不过3.0却换成了“滤镜”图形处理功能，“滤镜”不仅包括了透明度，而且还有模糊、扭曲、X光等CSS2里的各种滤镜效果
3.0 vs 2.0的跨越，以上提及的只是3分之1，剩下的篇幅问题不一一解释，请2.0老用户自行实验。
2.0的绘图、图形处理、动画、系统等功能栓引网友的一句玩笑“牛到家了！”，那么3.0则“牛出国了！”。
}

3.0相比2.0的代码体积的增长分析{
虽然3.0源文件相对2.0又增加了不少，但却进行了多方面的缩小与优化，以使加载时能“更快一点”、让界面“更小/好看一点”、让功能“更强/高效/完善一点”

原本HTML如“<span id='abc' style='width:100px'>”已被压缩成“<span id=abc style=width:100>”
“<input type=text **>”已被压缩成“<input ***>”，等等对界面显示标记的压缩
}
